End Generative Process Criterion

Max Simulation Time: An end criteria similar to max number of epochs, it will ensure that the MOSA process does not run indefinitely.

Minimal Temperature: An end criteria that will end a generative process if any objective function temperature drops below this value

Max number of epochs: Controls the depth of search, the higher the value the deeper the search should go.



MOSA Parameter Definitions:

Cooling Schedule These are the implemented cooling schedules which can control how Simulated Annealing switches from exploration to exploitative behaviors during the generative process.

  • Huang (default): HRSV Schedule which manipulates temperature based on objective function changes
  • Triki: Triki schedule which cools temperature based on objective function changes and the Triki Delta T hyperparameter you must specify in "Advanced Options".
  • Geometric: Basic schedule that always switches from explorative to exploitive over time through geometric cooling of temperatures using the Geometric Cooling Rate parameter foudn in "Advanced Options"

    • Acceptance Function: These are the implemented acceptance functions which can control how MOSA goes about accepting objectively worse solutions to explore a design space.

  • Standard (default): Exponential decrease analogous to traditional Simulated Annealing
  • Logistic: Logistic Curve Acceptance Probability Function
  • Linear: Scalar linear Acceptance Probability Function

    • Allow Self Intersecting Surfaces: This will allow the generative process to consider unobvious geometries to solve the objective functions.

  • Yes (default): Allow for a wider design space search using self intersecting surfaces
  • No: Do not allow self-intersecting surfaces

    • NT1: Number of iterations to initialize the temperatures (default value of 1000 recommended). Larger values can allow for a deeper initial archive.

    NT2: Number of accepted solutions in MOSA that controls when the temperature should be quenched (and the epoch counter increases)

    N_Bi: Number of iterations at some epoch i for which MOSA will execute a return to base to search the Pareto

    N_Bi Lower Limit: Lowest amount of iterations at some epoch to consider returning to base

    N_a: Similar to NT2, but it is a max number of accepted solutions at any given epoch before quenching the temperature

    r_b: Decreases N_Bi at each epoch iteration as we want to execute return to bases more often the longer the search goes to fully explore the Pareto

    r_i: Lowers the candidate set size at each epoch iteration such that we focus on more isolated regions of the Pareto the longer the search goes on

    Minimal Set Size: The minimal size of candidates we will randomly choose from when executing a return to base



    Design Constraint Definitions

    Max X: Max size of bounding box in X direction (in nanometers)

    Max Y: Max size of bounding box in Y direction (in nanometers)

    Max Z: Max size of bounding box in Z direction (in nanometers)

    Max Number of Nucleobases: During MOSA, all designs will comply to have an estimated scaffold length below this value. Note that this is just an estimate, and sometimes generated designs may have a slightly longer scaffold then desired!

    Minimal Allowable Edge Length: The smallest edge length allowable in a generated design, comes from DAEDALUS2 limitations

    Minimal Face Angle: The smallest face angle allowable in a generated design, comes from DAEDALUS2 limitations



    Advanced Option Definitions

    Seed: Random seed used in the MOSA process which allows for replication of results.

    Extend Rule Distance: Controls the distance of the normal face direction and extend vertex rules, in units of nm.

    Max Edge Multiplier: Controls the max edge length of a generated design as a multiple of the minimal allowable edge length

    Repulsion Constant: Sets the scale to the repulsion objective function calculation.

    Repulsion Distance: Controls the distance between any pair of two edges for which we should consider electrostatic repulsion.

    Max number of Edges per Node: Controls the max number of edges per node allowed during a search (as too many edges per node can lead to very flexible structures)

    Extend Rule Distance: Controls the distance of the normal face direction and extend vertex rules, in units of nm.

    Precision (number of decimals): Controls the number of decimals to use in the calculation.

    Cooling Rate (only if using Geometric): Sets the cooling rate for the geometric algorithm.

    Triki Delta T (only if using Triki): Specifies the delta T value when using the Triki algorithm.
