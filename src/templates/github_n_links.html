How to Use
1) You can use online create an input file to create an input file for the MOSA software (described below in 2). The input file is simply a JSON file with input conditions, and you may find it easier to just manipulate the JSON file once you use the software more than a few times.

2) Clone the MOSA code from GitHub here. You can simply run "MOSA_wireframe_generator JSON_INPUT_PATH OUTPUT_PATH" from terminal to let the generative process begin. When cloning from GitHub, we'd recommend setting up a new virtual environment and running "pip install requirements" so that the necessary Python packages are installed. You can read more on how to run MOSA from the GitHub repository webpage.

3) After running the MOSA optimizer, it will output a .aj1 file into the specified OUTPUT_PATH. You can upload this file in the Open Results File page and analyze the generated structures. From here you can download a desired structure, and the download will include: 1) a PLY file, 2) The input scaffold sequence, 3) a CanDo file, 4) a caDNAno file, and 5) oxDNA / oxView top and dat files. This package uses the DAEDALUS2 algorithm to automatically generate the scaffold and staple sequences on the generated structures PLY file using the standard M13 scaffold (by default) or through your own custom scaffold sequence.

Common Questions:
Q1: Why is MOSA not generating the output CanDo / caDNAno / oxView files?
A1: If the scaffold length you input is too short, the DAEDALUS2 software will fail to generate the wireframe structure.

Known Problems
P1: We are currently estimating the scaffold used through a simple process, and this is typically a conservative estimate. In reality, the scaffold required may be longer than what we estimate which could cause issues.
P2: We have seen DAEDALUS2 sometimes "scale up" edge lengths when not intended. We can not control this behavior, but we are looking to include design constraint checks which prevent these types of geometries from being evaluated.
